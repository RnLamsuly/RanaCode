# 开发文档 (针对 Vibe Coding Agent)

## 0. 项目概述
构建一个纯静态网页应用，用于展示开发者的算法题题解。  
首页设有两个入口：左侧进入“LinK”题单的题解列表，右侧进入“JD”题单的题解列表。  
点击入口跳转至对应题解列表页，列表中的每一项点击可展开查看完整题解。

题解以 Markdown 格式存储（.md 文件），每个文件对应一道题。  
列表页需自动读取所在目录下的全部 .md 文件，将文件名整理为列表展示。  
列表项展开后需要渲染并展示 Markdown 内容（支持嵌套 Markdown 渲染）。

## 2. 全局美术与样式约束

- **禁止使用任何 Emoji** （包括但不限于 💡 🎯 📊）。
- 页面背景：使用 `./image/rana2.jpg`（或 `image` 目录下其他图片）作为整页背景。
- 内容容器：使用 **半透明亚克力材质** 效果（`backdrop-filter: blur()` 配合半透明背景色），圆角边框，并添加适当的过渡动效或微交互。
- Logo 来源：**必须严格使用** `./Sample/RanaBPM/logo.png` 作为网站 Logo，不得更改。
- 美术风格参考：可参考 `./Sample/RanaBPM` 中的整体视觉设计（布局、色调、质感）。

## 3. 页面详细规格

### 3.1 主页 (`index.html`)

**布局**
- 页面分上下两大区域：
  - 上部区域：独占一行，显示 **动态时间** 和 **问候语**（具体实现参考 `./Sample/RanaBPM/index.html` 的时间与问候逻辑）。
  - 下部区域：水平平分为左右两部分。
    - 左侧区域：作为“LinK题单”的入口，点击跳转至 `./LinK/LinKList.html`。
    - 右侧区域：作为“JD题单”的入口，点击跳转至 `./JD/JDList.html`。

**入口交互**
- 左右两个入口设计为卡片样式，使用亚克力材质，悬停时产生视觉反馈。
- 入口卡片内应包含题单标题及简略引导文字，不得使用 Emoji。

### 3.2 题解列表页面 (`LinKList.html` 与 `JDList.html`)

两个题解列表页面结构完全相同，仅对应不同的目录和资源。  
以 LinKList 为例，JDList 同理替换路径。

**整体布局**
- 页面分为左右两栏：
  - 左侧栏宽度占 **2/3**。
  - 右侧栏宽度占 **1/3**。

#### 3.2.1 左侧列表区域
- 自动读取当前页面所在目录（`./LinK/` 或 `./JD/`）下的全部 `.md` 文件。
- 将文件名（不含扩展名）按 **字典序升序** 排列，每一项生成一个长条形卡片。
- 卡片内容：
  - 左侧显示文件名（视为题目标题）。
  - 右侧包含一个 **“展开”按钮**。
- 点击“展开”后：
  - 该卡片向下平滑展开一个区域，用于展示详细题解。
  - 同时通过异步方式获取对应的 `.md` 文件内容，并使用 Markdown 渲染器（如 marked.js）将其转换为 HTML 后插入展开区域。
  - 展开区域应支持嵌套 Markdown 的正常解析与展示。
- 再次点击或使用关闭按钮可折叠展开区域。

#### 3.2.2 右侧搜索区域
- 包含一个文本输入框，作为搜索栏。
- 用户输入子串时，**实时过滤**左侧列表项，只显示文件名中包含该子串（不区分大小写）的卡片。
- 搜索时保持列表顺序不变，仅控制显示/隐藏。

## 4. 技术要求与实现提示

- 所有页面为静态 HTML/CSS/JS，不需后端。
- 使用原生 JavaScript 或轻量工具库完成功能，不引入大型框架（除非必要）。
- Markdown 渲染可使用 CDN 引入 `marked` 库，或自行实现一个简易渲染器。
- 动态获取 .md 文件列表：由于静态页无法直接读取服务器目录，需设计合理方案（例如：部署时通过脚本生成一个包含所有 .md 文件名的列表文件，页面加载时 fetch 该列表；或利用服务器目录索引）。请保证最终列表页面能够正确显示目录中所有题解文件。
- 页面样式需符合第 2 节美术约束，所有动效需流畅，亚克力效果需兼容主流浏览器。
- 时间与问候逻辑务必参考 `./Sample/RanaBPM/index.html` 的实现方式，确保行为一致。
- 不可出现任何 Emoji。
