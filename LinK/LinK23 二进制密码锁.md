# LinK23 二进制密码锁
[题目链接](http://xmuoj.com/contest/362/problem/LinK23)

## 思路

先考虑密码锁的第 $1$ 位，其受到两个按钮的控制：第 $1$ 个与第 $2$ 个。

假设第 $1$ 个按钮是否按下**已经确定**。为了使第 $1$ 位转换为目标，我们能够确定第 $2$ 位是否要按下。

接着我们考虑第 $2$ 位。第 $2$ 位受到按钮 $1$ 、 $2$ 、 $3$ 的控制。其中 $1$ 与 $2$ 是否按下已经确定。为了让第 $2$ 位转换为目标，我们能够确定第 $3$ 位是否要按下。

以此类推，我们根据第 $n-1$ 位确定第 $n$ 个按钮是否按下后，第 $n$ 个按钮的状态也就确定了。比较第 $n$ 个按钮与目标是否一致便可以判断方案的正确性。

那么我们如何确定第 $1$ 个按钮是否要按下呢？事实上，我们没有办法预先知道第 $1$ 位是否要按下，但是我们可以枚举第 $1$ 位按下与不按下的两种情况。

---

## 处理细节

将初始串与目标串对比会导致代码复杂而臃肿。我们可以采取如下处理：

仅记录初始串与目标串的**差异**，即记录每一位**需不需要被改变**。我们可以通过求两个二进制串的**异或**来得到差异。

---

## 注意

澄清一个误区：第一位按下与不按下两种情况中，**并非**有且仅有一种情况有解。

即：会出现**有两解**的情况，也会有**无解**的情况。一定要单独判断。

---

## AC代码
```cpp
#include <bits/stdc++.h>
 
std::string A,B; // 字符串储存输入
bool a[50],b[50]; //a,b数组均记录差异，但a数组代表第一位按下的情况，b反之
 
int main()
{
	std::cin>>A>>B;
	int n=A.length();
	int ansa=1,ansb=0; //ans记录按下按钮的次数，两个ans对应两种情况
	for(int i=0;i<n;++i)
	{
		a[i]=A[i]-'0'; //将字符转换为布尔值，下同
		b[i]=B[i]-'0'; //这里临时用b处理B串
		a[i]=a[i]^b[i]; //异或求差异
		b[i]=a[i]; //将结果复制给b
	}
	a[0]=!a[0]; //模拟按下第一位的效果
	a[1]=!a[1];
	for(int i=0;i<n-1;++i) //从第二位开始判断按钮是否需要被按下
	{
		if(a[i]==true)
		{
			a[i]=!a[i];
			a[i+1]=!a[i+1];
			a[i+2]=!a[i+2];
			ansa++;
		}
		if(b[i]==true)
		{
			b[i]=!b[i];
			b[i+1]=!b[i+1];
			b[i+2]=!b[i+2];
			ansb++;
		}
	}
    //分析解的情况
	if(a[n-1] && b[n-1])std::cout<<"impossible";
	if(a[n-1] && !b[n-1])std::cout<<ansb;
	if(!a[n-1] && b[n-1])std::cout<<ansa;
	if(!a[n-1] && !b[n-1])std::cout<<(ansa<ansb)?ansa:ansb;
	return 0;
	
}
```
