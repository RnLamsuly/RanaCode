# LinK33 攻击范围
[题目链接](http://xmuoj.com/contest/362/problem/LinK33)

## 思路

这题依旧是[二分查找](https://oi-wiki.org/basic/binary/)，不过查询对象从**点**变为了**区间**。

对于区间，我们只需要查找出左端点以及右端点即可。

由于目标值**不包含于目标左端点的左侧**，也**不包含于目标右端点的右侧**，根据上述规则可以分别二分查找目标左右端点。

## 拓展：二进制优化二分查找

二进制优化二分查找的本质是：将二分查找中的折半策略，替换为**从大到小尝试 2 的幂次跳跃**。

它的理论基础是：任何一个整数都可以表示为若干个 2 的幂次之和。

例如：$13 = 8 + 4 + 1$

因此，我们可以从最大的步长开始尝试，如果跳过去仍满足条件，就跳过去；否则尝试更小的步长。最终一定能精确到达目标位置。

我们使用**位运算**来优化二分查找，可以**避免除法**。由于除法的常数**远大于**位运算，故使用二进制优化可以一定程度上**提高运行效率**。

此外，使用二进制优化不需要预先知道待查询数组长度，并且可以避免陷入死循环。

## AC代码
```cpp
#include <iostream>
const int MAXN=1e5+5;
int a[MAXN];
 
int main()
{
	int n,q;
	scanf("%d%d",&n,&q);
	for(int i=0;i<n;++i)scanf("%d",&a[i]);
	while(q--)
	{
		int tar;
		scanf("%d",&tar);
		int l=-1,r=-1; //l和r分别是目标区间的左端点和右端点，分别进行查找
		for(int base=1<<20;base;base>>=1)
		{
			if(l+base<n && a[l+base]<tar)l+=base;
			if(r+base<n && a[r+base]<=tar)r+=base;
		}
		if(r==l)printf("-1 -1\n"); //[l,r)中l==r，说明没有结果
		else printf("%d %d\n",l+1,r);
	}
	return 0;
}
```
